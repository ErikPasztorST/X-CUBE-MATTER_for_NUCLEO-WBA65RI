Release Notes for X-CUBE-MATTER

Copyright © 2025 STMicroelectronics

Purpose

  • This expansion pakage provides Matter over Thread connectivity.

  • Applications with and without OTA support are provided for STM32WB and STM32WBA DK boards.

  • Note that SBSFU (Secure Boot Secure Firmware Update) used for secure boot and OTA is based on X-CUBE-SBSFU with an added feature: the external flash memory support. SBSFU is applicable to STM32WB only. It is used to store NVM, M0 and M4 binaries for OTA (firmware update Over The Air). See Matter wiki (link below) for details about SBSFU external flash usage.

  • Note that OEMiRoT used for secure boot and OTA is based on example provided into official CubeFW STM32WBA. OEMiRoT is applicable to STM32WBA only.

  • Note that in Utilities\ folder there is an APK\ directory that contains Chiptool .apk built from CSA source files. Chiptool android apk is delivered because not straightforward for the customer to build it from ConnectedHomeIP source files

  • Projects Release Note is here: \Projects

  • Wireless Copro Release Note (only applicable to STM32WB) is here: \Projects\STM32WB_Copro_Wireless_Binaries\STM32WB5x

Here is the list of references to the user documents:

  • Matter STMicroelectronics MCU Wiki - ST Wiki dedicated to MCU, Matter page
  • DB5144 - Data Brief of the Matter software expansion for STM32Cube

Known limitations

with WBA6, following limitations identified:

  • Reset Factory: WBA6, after a reset factory, user has to unplug and plug the board (more information about this on Matter Wiki).
  • Low power mode is not enabled with Generic Switch application
  • ITS does not support isolation based on TrustZone
  • Only Non-Secure Applicaion can be updated OTA. OEMiROT Secure App cannot be updated

Known limitations

  • Due to QSPI limitations, external Flash clock was lowered to 12.8 MHz.
  • Only D version of MB1292 (STM32WB5MM-DK) is supported (B version is not).
  • Only CubeIDE is supported (neither IAR/EWARM nor Keil/MDK-ARM are provided) in this first version.
  • In this first version STM32CubeMX code generation is not supported.
  • Factory Data are all stored in external flash (to store private key associated to DAC in secure storage contact STMicroelectronics to get the needed firmware).
  • The Factory Data are not yet write protected.
  • For SBSFU only development mode configuration has been tested (production mode tests on going). See AN5056 Chapter production mode configuration.
  • BLE_ota provided for evaluation purpose only.

Development toolchains and compilers

  • STM32CubeIDE v1.14.1 (GNU Tools for STM32 (11.3.rel1))
  • Internal pre-certifications has not been done with CubeIDE 1.14.1 but with older version see Project Release_Notes for details.

Supported devices and boards

  • STM32WB5MM-DK : Discovery kit with STM32WB5MMG MCU, minimum Rev D
  • P-NUCLEO-WB55.Nucleo : Bluetooth 5 and 802.15.4 Nucleo Pack including USB dongle and Nucleo-64 with STM32WB55 MCUs, supports Arduino Uno V3 and ST morpho connectivity
